# Web Ad Radar 最终实施报告

日期：2026-06-05

## 一、实施目标

本次实施将 `web-ad-radar` 调整为可供 OpenClaw 定时调用的独立 Skill。OpenClaw 后续负责将结构化结果写入飞书多维表格，本 Skill 负责稳定完成职位抓取、清洗、标签、雇主分析和中文 Markdown 报告输出。

## 二、核心规则

- 默认运行日期：未传入 `--from/--to` 时，默认抓取 Asia/Shanghai 时区的前一天。
- 日期可识别来源：按发布日期或更新日期过滤，只保留目标日期范围内的职位，并通过分页继续检查。
- 日期不可识别来源：不再把抓取当天伪装成发布日期；改为分页抓取最多 30 个当前可见职位。
- Hays：按无日期来源策略抓取前 3 页，目标 30 个职位。
- 报告字段：职位名称和 JD 分列；发布日期和抓取日期分列，避免混淆。
- 结构化存储：SQLite `jobs` 表继续保留稳定字段，包括 `source_slug/source_name/title/url/location/published_at/first_seen_at/last_seen_at/jd_text/function_label/industry_label/employer_guess` 相关信息。

## 八、已知限制

- 当前大多数猎头站点不公开真实发布日期，因此这些来源按“当前可见职位样本”抓取 30 条，后续由 OpenClaw/飞书侧做去重和历史差分。
- Risfond 是日期型来源，能按发布日期过滤；若目标日期无职位，会正常输出 0 条而不是补抓旧职位。
- 本 Skill 暂不直接写入飞书；OpenClaw 后续可基于 SQLite 或 Markdown 表格字段进行二次开发。
